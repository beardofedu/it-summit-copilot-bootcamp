# Lab 02 — Prompt Engineering for Better Copilot Results

This lab focuses on a simple truth: **Copilot usually improves when your prompt improves**.

## Goal

Practice turning vague comments into useful prompts that include:
- context
- intent
- constraints

## Files in this lab

- `PROMPTING_GUIDE.md` — reference guide for better prompting
- `exercises/bad_prompts.py` — intentionally weak prompts to improve
- `exercises/improved_prompts.py` — reference examples after you try the exercise yourself

## Recommended workflow

1. Read `PROMPTING_GUIDE.md`
2. Open `exercises/bad_prompts.py`
3. Rewrite each comment to be clearer and more actionable
4. Try generating code after your improved prompt
5. Compare your work with `exercises/improved_prompts.py`

## What success looks like

By the end of this lab, you should be able to:
- spot vague prompts quickly
- rewrite them using better context and constraints
- explain why your rewrite should lead to better Copilot output

## Coaching question

For every prompt, ask:

> If a new teammate read this comment, would they know what the code is supposed to do?

If the answer is “not really,” Copilot probably will not either.
